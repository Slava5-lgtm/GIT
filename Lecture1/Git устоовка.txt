Введение
Системы контроля версий, такие как Git, необходимы для современных передовых практик разработки программного обеспечения. Управление версиями позволяет отслеживать ваше программное обеспечение на уровне исходного кода. Вы можете отслеживать изменения, возвращаться к предыдущим этапам и создавать альтернативные версии файлов и каталогов.

Файлы многих программных проектов хранятся в репозиториях Git, а такие платформы, как GitHub, GitLab и Bitbucket, помогают облегчить совместное использование проектов разработки программного обеспечения и совместную работу.

В этом руководстве мы рассмотрим, как установить и настроить Git на сервере Ubuntu 20.04. Мы рассмотрим, как установить программное обеспечение двумя различными способами: через встроенный менеджер пакетов и через исходный код. Каждый из этих подходов имеет свои преимущества в зависимости от ваших конкретных потребностей.

Упростите развертывание приложений с помощью DigitalOcean App Platform. Развертывание непосредственно с GitHub занимает считанные минуты.

Предварительные требования
Вам понадобится сервер Ubuntu 20.04 с учетной записью суперпользователя, не являющегося суперпользователем root.

Чтобы настроить это, вы можете следовать нашему руководству по начальной настройке сервера для Ubuntu 20.04.

Настроив сервер и пользователя, вы готовы начать.

Установка Git с пакетами по умолчанию
Вариант установки с пакетами по умолчанию лучше всего подходит, если вы хотите быстро освоиться с Git, если вы предпочитаете широко используемую стабильную версию или если вы не ищете новейшие доступные функции. Если вы ищете самую последнюю версию, вам следует перейти к разделу о установке из исходного кода.

Скорее всего, Git уже установлен на вашем сервере Ubuntu 20.04. Вы можете подтвердить, что это так на вашем сервере с помощью следующей команды:

git --version
Если вы получаете вывод, подобный следующему, значит, Git уже установлен.

Output
git version 2.25.1
Если это вас устраивает, то вы можете перейти к настройке Git или прочитать следующий раздел о том, как установить из исходного кода, если вам нужна более актуальная версия.

Однако, если вы не получили номер версии Git, вы можете установить его с помощью Ubuntu default package manager APT.

Сначала используйте инструменты управления пакетами apt для обновления индекса вашего локального пакета.

sudo apt update
После завершения обновления вы можете установить Git:

sudo apt install git
Вы можете подтвердить правильность установки Git, выполнив следующую команду и убедившись, что получаете соответствующие выходные данные.

git --version
Output
git version 2.25.1
Теперь, когда Git успешно установлен, вы можете перейти к разделу этого руководства "Настройка Git", чтобы завершить настройку.

Установка Git из исходного кода
Если вы ищете более гибкий метод установки Git, возможно, вам захочется скомпилировать программное обеспечение из исходного кода, который мы рассмотрим в этом разделе. Это займет больше времени и не будет поддерживаться через ваш менеджер пакетов, но это позволит вам загрузить последнюю версию и даст вам больший контроль над включаемыми вами параметрами, если вы хотите внести изменения.

Проверьте версию Git, установленную в данный момент на сервере:

git --version
Если Git установлен, вы получите вывод, подобный следующему:

Output
git version 2.25.1
Прежде чем вы начнете, вам необходимо установить программное обеспечение, от которого зависит Git. Все это доступно в репозиториях по умолчанию, поэтому мы можем обновить наш локальный индекс пакета, а затем установить соответствующие пакеты.

sudo apt update
sudo apt install libz-dev libssl-dev libcurl4-gnutls-dev libexpat1-dev gettext cmake gcc
После того, как вы установили необходимые зависимости, создайте временный каталог и переместитесь в него. Именно сюда мы загрузим наш архив Git.

mkdir tmp
cd /tmp
С веб-сайта проекта Git мы можем перейти к списку архивов, доступному по адресу https://mirrors.edge.kernel.org/pub/software/scm/git/ и загрузить желаемую версию. На момент написания статьи самой последней версией является 2.26.2, поэтому мы загрузим ее в демонстрационных целях. Мы будем использовать curl и выводить файл, в который мы загружаемgit.tar.gz.

curl -o git.tar.gz https://mirrors.edge.kernel.org/pub/software/scm/git/git-2.26.2.tar.gz
Распакуйте сжатый файл tarball:

tar -zxf git.tar.gz
Далее перейдите в новый каталог Git:

cd git-*
Теперь вы можете создать пакет и установить его, введя эти две команды:

make prefix=/usr/local all
sudo make prefix=/usr/local install
Теперь замените процесс оболочки, чтобы использовалась только что установленная нами версия Git:

 exec bash
Завершив это, вы можете быть уверены, что ваша установка прошла успешно, проверив версию.

git --version
Output
git version 2.26.2
Теперь, когда Git успешно установлен, вы можете завершить настройку.

Настройка Git
После того, как вы будете удовлетворены своей версией Git, вам следует настроить Git таким образом, чтобы создаваемые вами сообщения о фиксации содержали правильную информацию и поддерживали вас при создании вашего программного проекта.

Настройка может быть достигнута с помощью git config команды. В частности, нам нужно указать наше имя и адрес электронной почты, потому что Git встраивает эту информацию в каждый выполняемый нами коммит. Мы можем продолжить и добавить эту информацию, введя:

git config --global user.name "Your Name"
git config --global user.email "youremail@domain.com"
Мы можем отобразить все элементы конфигурации, которые были установлены, введя:

git config --list
Output
user.name=Your Name
user.email=youremail@domain.com
...
Введенная вами информация сохраняется в вашем файле конфигурации Git, который вы можете дополнительно отредактировать вручную с помощью текстового редактора по вашему выбору, например, этого (мы будем использовать nano):

nano ~/.gitconfig
~/.содержимое gitconfig
[user]
  name = Your Name
  email = youremail@domain.com
Нажмите CTRL и X, затем Y затем ENTER, чтобы выйти из текстового редактора.

Есть много других параметров, которые вы можете установить, но это два основных из них. Если вы пропустите этот шаг, вы, вероятно, увидите предупреждения при фиксации в Git. Это усложнит вам работу, потому что затем вам придется пересмотреть сделанные вами коммиты с исправленной информацией.
